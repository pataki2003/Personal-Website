Frontend Project

Portfolio Website

A custom-built personal portfolio website redesigned from scratch to present my skills, projects, and technical direction in a cleaner, more modern, and more structured way.

Status: In Progress Focus: Frontend Development Type: Personal Brand Website

The Problem

The earlier version of the portfolio felt too generic and visually flat. It worked on a basic level, but it did not communicate much character, structure, or technical direction.

A portfolio should not just list information. It should create a stronger first impression, guide attention well, and reflect the way the developer thinks about design and presentation.

The Solution

I rebuilt the portfolio with custom HTML and CSS instead of relying on a framework-heavy layout. The goal was to create a cleaner visual hierarchy, stronger spacing, a more modern interface, and a more polished overall feel.

I also introduced a dark and light theme system to make the interface more flexible and improve the overall user experience across different viewing preferences.

Key Features

  • Fully custom HTML and CSS structure
  • Responsive layout across screen sizes
  • Dark and light theme toggle
  • Reusable project detail page system
  • Cleaner visual hierarchy and spacing

Challenges and Thinking

One of the biggest challenges was turning a simple portfolio into something that feels more intentional without making it visually overloaded. It needed to stay clean, but not boring. Modern, but not artificial.

The redesign focused on balancing readability, stronger branding, and a more structured content flow while still keeping the codebase simple and easy to extend later.

Outcome

This project helped me improve my frontend thinking, design structure, and ability to create a more polished presentation layer without depending on a ready-made visual framework.

It also created a stronger foundation for future additions like animations, project interactions, and more advanced JavaScript functionality.
